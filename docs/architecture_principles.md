# アーキテクチャ進化方針

## 1. この文書の目的

このプロジェクトの最小エナクティブエージェントは、単なる PoC ではない。
将来より複雑なエージェンシーへ連続的に発展しうるアーキテクチャの**原型（body plan）**である。

生物の進化が示すように、基本体制は一度確立されると以後の発展を強く制約する。
脊椎動物は節足動物にならないし、四肢が六肢に変わることはない。
内耳のように、既存構造の増築と転用が進化の基本形である。

この文書は、アーキテクチャの拡張において「何を変えてよく、何を変えてはならないか」を定める。
今後の実験・実装の判断は、この方針に照らして行う。

---

## 2. 現在の基本体制

```
i_t（環境との感覚結合）→ h_t（遅い内部状態）→ m_t（行動モード）→ a_t（行動）
                      ↑                                              |
                      └──────── 環境 ←────────────────────────────────┘
```

### 各要素の役割分担

| 要素 | 体制上の役割 | 性質 |
|---|---|---|
| **h**（遅い内部状態）| 内臓的・時間的な持続状態 | 枯渇圧、探索ドリフトなど。身体の内部事情を反映する |
| **m**（行動モード）| 行動レジームの選択と維持 | 活用/探索/回避など。「何をしているか」の体制的状態 |
| **i**（環境入力）| 身体-環境結合の産物 | 局所的な感覚信号。エージェントの身体と環境の関係から生じる |
| **a**（行動出力）| 環境への働きかけ | 移動、旋回など。環境を変え、次の i を変える |
| **b_h**（バイアス）| 基礎駆動 | 入力がなくても h を駆動する恒常的な圧力。代謝的駆動の原基 |
| **閉ループ**| 全体の存在条件 | エージェンシーは内部計算ではなく、このループの性質として存在する |

---

## 3. 拡張の原則

### 原則 1: 役割分担を保つ

h は内臓的・時間的、m は行動的・体制的、i は感覚的・結合的。
この分担は将来の拡張でも維持する。

**許容される拡張**: h の次元を増やす（多時間スケール化）、m のモード数を増やす（行動レパートリーの拡大）、i のチャンネルを増やす（感覚モダリティの追加）。

**禁止される混合**: h に空間記憶を混ぜる（内臓系に空間認知を担わせる）、m に内部状態の役割を持たせる（行動体制と内部事情の融合）、i に内部状態を含める（感覚と自己状態の区別の喪失）。

### 原則 2: 増築であって改造ではない

既存の構造は維持し、新しい能力は新しい経路の追加で実現する。

**許容される増築**:
- h と m の間に新しい中間変数を挿入する
- i に新しいセンサーチャンネルを追加する
- 既存の状態変数から新しい出力経路を引く（予測モジュールなど）

**禁止される改造**:
- h の更新式を根本的に変える（leaky integration の廃止など）
- 閉ループ構造を破る（エージェントが環境全状態を直接参照するなど）
- m を介さず h から直接行動を生成する経路を主経路にする

### 原則 3: 新規変数は独立した役割で導入する

新しい状態変数を追加する場合、既存変数とは異なる明確な役割を持たせる。

**例**: 空間的な情報が必要になった場合
- (良) 新しい状態変数 s_t を「身体の空間的状態」として導入し、h とは独立に管理する
- (良) i_t を豊かにして空間勾配を含める（感覚結合の強化）
- (悪) h_t に空間次元を追加する（h の内臓的役割が変質する）

### 原則 4: 感覚は身体-環境結合として扱う

観測はエージェントの能動的行為である。環境が観測を「与える」のではなく、
エージェントが環境を「感覚する」。

この原則は、将来の能動的センシング（どこを見るか、注意の配分）への発展を確保するため、
実装レベルでも感覚計算をエージェント側に置く方向へ整備する。

### 原則 5: 報酬信号を導入しない

このプロジェクトの核心は「外部目標なしにコヒーレントな行動が生じる」ことである。
スカラー報酬信号はこの核心を損なう。

行動の評価は外部の評価者（eval.py）が事後的に行う。
エージェント自身は報酬を参照しない。これは設計方針であり、制約ではない。

将来、内発的な価値信号が必要になった場合は、それは報酬ではなく
「コヒーレンス駆動の内部表現」として、h の力学の中に位置づける。

## 4. 拡張のロードマップ

現在の体制から、段階的に以下の拡張が考えられる。
各段階で体制の基本は保持される。

### 段階 1: 現在の体制内の深化（体制不変）

- パラメータ感度分析
- 履歴依存性の直接検証
- より豊かな評価指標

### 段階 2: 感覚結合の強化（i の拡張）

- 空間勾配を i に追加
- 複数感覚チャンネル
- 観測インターフェースのエージェント側移動

### 段階 3: 環境の複雑化（体制不変、環境変更）

- 複数食物パッチ
- 動的リスク
- 時間変動する環境

### 段階 4: 内部構造の増築

- h の多時間スケール化（h_slow, h_fast）
- 予測モジュールの追加（h, m から次の i を予測し、誤差を m に注入）
- m の階層化（低次モードと高次モード）

### 段階 5: 学習の導入

- パラメータ探索（GA 等）
- コヒーレンス駆動に基づく内発的学習則
- ※ 外部報酬最大化ではなく、存続可能性と探索連続性に基づく

## 5. 判断基準

新しい実験や実装の変更を行う際、以下を問う。

1. **h/m/i/a の役割分担は保たれるか？**
2. **既存の構造を壊していないか？増築か改造か？**
3. **新しい変数は既存変数と重複しない独自の役割を持つか？**
4. **閉ループ構造は維持されるか？**
5. **この変更を元に戻さずに、さらに先に進めるか？**

5 番目が特に重要である。「この一手を打った後、次の一手の選択肢は広がるか、狭まるか」を常に問う。
